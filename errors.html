Tutorial: CHAOS.Portal Javascript Client

Error handling

The CHAOS.Portal Javascript Client is a communication client for CHAOS.Portal
CHAOS.Portal is the web API for CHAOS.

Error handling

Let's add some error handling code to that call. 

client.SessionAcquired().Add(function(sender, sessionGUID) {
  client.Object_GetBySearch(
    showObjects                     // callback
    , "*"                         // query
    , "5906a41b-feae-48db-bfb7-714b3e105396" // schemas
    , "da"                          // langCode
    , null                          // sort
    , ChaosSettings.accessPointGUID // accessPointGuid
    , 0                             // pageIndex
    , 3                             // pageSize
  );
});

function showObjects(serviceResult) {
  if (serviceResult.WasSuccess() && serviceResult.MCM().WasSuccess()) {
    var count = serviceResult.MCM().Count();
    var totalcount = serviceResult.MCM().TotalCount();
    alert("Got " + count + "/" + totalcount + " objects");
    console.log(serviceResult.MCM().Results());
  } else {
    if (!serviceResult.WasSuccess()) {
      alert("Portal error: " + serviceResult.Error());
      console.log(serviceResult.Error());
    } else if (!serviceResult.MCM().WasSuccess()) {
      alert("MCM error: " + serviceResult.MCM().Error());
      console.log(serviceResult.MCM().Error());
    }
  }
}

Here we try searching for * -- which is everything. This is not allowed on this particular field in the Solr index (you can assume that this is the case for most fields). Running this code you should get: MCM error: The remote server returned an error: (400) Bad Request. This means that there was an error in the query to the Solr index.

In general CHAOS.Portal can give quite a large range of errors. Giving an accessPointGUID that is too short or too long typically results in a System.ArgumentOutOfRangeException: Index and length must refer to a location within the string. But if the accessPointGUID contains invalid characters (non-hex: 0-F) it returns: System.FormatException: Could not find any recognizable digits.

In other words, the CHAOS.Portal error handling is not complete yet.
A basic rule of thumb is to check whether TotalCount() is null. This is usually indicates an error.
